# Dimension 2 — Traceability Completeness

This dimension audited whether every Functional Requirement (FR) and Acceptance Criterion (AC) defined in `spec/01-requirements/` is correctly and currently wired into `traceability-matrix.csv`, and whether the CFG-, DATA-, PERM-, and UI- (Surface) identifiers cited in matrix rows actually resolve to real, registered ids in `config-registry.md`, `schema.md`, `PERMISSION_NODES.md`, and `spec/03-surfaces/`. Checks included: cross-referencing FR ids present in each component file against the rows present in the matrix (looking for missing rows after change-control additions); comparing each component file's self-reported FR-count header/footer against its actual body content; validating that cited PERM- ids exist verbatim in the permission catalog and match the correct authority tier where an OD governs the split; validating that cited CFG-/DATA- ids exist verbatim in their registries; validating that cited UI- surface ids match the canonical Surface IDs minted in Phase 3; and spot-checking citation-prefix precision (FR- vs AC- for sub-numbered acceptance criteria). This is the working evidence file (pre-adversarial-verification) — findings below are rendered as reported by the dimension's agents, without editorializing on which will survive verification.
