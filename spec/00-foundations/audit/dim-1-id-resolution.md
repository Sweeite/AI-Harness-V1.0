# Dimension 1 — Cross-phase ID Resolution

## Summary

This dimension checked cross-phase ID resolution across the repo: whether every cited ID (FR-, CFG-/config-registry keys, DATA- table/column citations, PERM- permission nodes, and UI- surface IDs) actually resolves to a real, minted definition elsewhere in the spec, or whether it dangles. Agents grepped source-of-truth locations for each ID family (component requirement files for FR-, `spec/02-config/config-registry.md` for config keys, `spec/04-data-model/schema.md` for DATA- tables/columns, `spec/01-requirements/component-01-rbac.md`'s PERM catalog for permission nodes, and `spec/03-surfaces/*.md` for UI- surface IDs) and cross-referenced citing files (`spec/02-config/_HARVEST.md`, `traceability-matrix.csv`, `spec/04-data-model/rls-policies.md`, `spec/01-requirements/component-00-login.md`, `component-01-rbac.md`, `component-03-tool-layer.md`, tool-integration dossiers, and surface files) to confirm whether each citation resolves, is a naming-drift false positive, or is a genuine dangling/unminted reference.
