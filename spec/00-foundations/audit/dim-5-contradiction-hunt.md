# Dimension 5 — Contradiction Hunt

## Summary

This dimension's agents hunted for internal contradictions across the AI-Harness-V1.0 spec repository: cases where two or more Approved/locked artifacts (ADRs, FRs, ACs, config-registry rows, schema DDL, surface specs, and foundational audit docs such as PERMISSION_NODES.md and what-makes-it-great.md) assert mutually incompatible things about the same rule, key, mechanism, or count — stale cross-references to superseded or wrong FR/AC IDs, config keys renamed or restructured in the built registry without a change-control note reconciling the locked ADR that mandated them, locked hard limits silently loosened by a later component FR, self-referential count/tally errors (FR counts, node counts, category counts, audit-sink counts) in a file's own status header or narrative, missing schema artifacts (tables/columns/constraints) that a locked ADR or Approved FR requires to exist, and surfaces citing the wrong section/PERM-gate/AC family for a given config key or behavior. Every row below is a **raw, pre-verification finding** exactly as reported by the dimension's agents — nothing has been adjudicated, deduplicated across dimensions, or filtered by confidence beyond what the source agent itself noted (some entries carry an agent-supplied `verdict` tag such as CONFIRMED or PLAUSIBLE; where present, this is folded into the Recommendation column and should not be read as independent editorial verification).
