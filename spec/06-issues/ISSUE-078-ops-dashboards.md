# ISSUE-078 — Ops dashboards (single-deployment + super-admin fleet console)

> **Self-sufficiency contract (read this first).** This issue is a *complete, precise build
> order that points into the repo by ID*. It does **not** restate `AC-*` text — that lives in the
> FR and is read there (copying it would create a second source of truth that rots = Rule-0
> violation). A builder with **zero conversation history** must be able to open the files named in
> the Context manifest and build this slice to its Definition of done **without guessing**.

## 1. Goal (one line)
Build the two operator dashboards that *render* C7/C10 observability signals — the per-deployment **Operations dashboard** (surface-05, nine polled panels) and the cross-deployment **Super-Admin fleet console** (surface-06, fleet grid + seven management sections) — each honest about its own freshness and never rendering a false-healthy view (#3).

## 2. Scope — in / out
**In:**
- **surface-05 (Operations dashboard)** — the single-deployment instrument panel: the nine polled, individually RBAC-scoped panels (System Health · Failure Health · Connector Health · Memory Health · Event Log · Dead-Letter Queue · Cost · Guardrail Log · Self-Improvement), each rendering its home component's already-produced signal over the C7 poll transport, each showing last-updated-at + a stale/error state that never masquerades as a healthy "0"/"✓" (FR-7.VIEW.001, the polling side of FR-7.RTP.002/004).
- **surface-06 (Super-Admin fleet console)** — the sole cross-deployment surface, on the separate management deployment (ADR-001 §7): the Fleet Health Grid landing + sections B–H (Cross-Deployment Alerts · Releases & CI/CD · Migrations · Provisioning & Onboarding · Cross-Deployment Cost · Backup Health · Client Registry & Offboarding), rendered **from the push-fed management store only, never a client pull** (FR-7.MGM.003–005, FR-7.VIEW.002), with the frozen-≠-dead and stale/unreachable rendering that upholds #2 (map-not-warehouse) and #3.
- The **per-panel / per-section RBAC gating** at render (an unpermitted panel is *absent*, not empty — AC-7.VIEW.002.1), consuming the C1 nodes and the surface-minted nodes (`PERM-dashboard.ops`, Operations-Actions nodes OD-121/OD-167; the five `PERM-fleet.*` nodes OD-125).
- The **offboarding guarded multi-step wizard** UI on surface-06 §H (OD-127) — rendering the C10 workflow's #1 gates (export-verified-before-delete, sign-off-before-freeze, inline two-person auth on hard-delete) as enforced UI steps; the workflow *logic* is owned by C10 issues, this renders + gates it.

**Out:**
- The **producing FRs behind every panel** — event_log/retention/redaction (C7 LOG → ISSUE-011/010/077), the real-time/polling contract + connection budget (C7 RTP → **ISSUE-076**), alerting + escalation + notification centre + fails-loud (C7 ALR → **ISSUE-075**), cost meter + ladder signal (C7 COST → ISSUE-074), management-plane views + retention/export + feedback flywheel (C7 MGM/VIEW/OPT → **ISSUE-077**), and every home-component signal (C2 memory health, C3 connector health, C5 loops/queue/DLQ, C6 guardrail log, C8 agent health/drift, C9 Insight suggestions). This issue **renders** them; it builds no signal.
- The **Realtime** surfaces (approval queue surface-04, notification centre surface-07) — those are the *two* live surfaces (FR-7.RTP.001); these two dashboards are strictly polling.
- The **management-plane substrate + lifecycle logic** — `client_registry`/ingest/token lifecycle (C10 MGT → ISSUE-012), releases/migrations (C10 DEP/MIG → ISSUE-080/081), provisioning (C10 PRV → ISSUE-007), the offboarding *workflow execution* (C10 OFF → ISSUE-083), backup/DR (Phase 5 → ISSUE-085). This surface renders their status + gates their operator actions; it does not implement them.
- **Mobile** rendering of these summaries (surface-12 → ISSUE-079); the config editing of every threshold/cadence read here (surface-01 → ISSUE-086).
