# ISSUE-025 — Retrieval + ranking + clearance-before-ranking + answer modes

> **Self-sufficiency contract (read this first).** This issue is a *complete, precise build
> order that points into the repo by ID*. It does **not** restate `AC-*` text — that lives in the
> FR and is read there (copying it would create a second source of truth that rots = Rule-0
> violation). A builder with **zero conversation history** must be able to open the files named in
> the Context manifest and build this slice to its Definition of done **without guessing**.

## 1. Goal (one line)
Build the C2 read path: given a task, extract its entities, run dual (keyword + vector) search, apply the candidate filters, drop out-of-clearance/out-of-scope memories **before** ranking, rank + trim to the injection set, inject as Business Context (never auto-injecting Restricted), and emit the answer-mode / Retrieval-Sufficiency signals — the retrieval keystone that Layer-3 injection (ISSUE-045), the run pipeline (ISSUE-053), agent scoping (ISSUE-063), and the proactive generators (ISSUE-069) all consume.

## 2. Scope — in / out
**In:** The C2 **RET** area group end-to-end — entity extraction from the incoming task; the two-arm dual search (entity-scoped keyword arm + top-~20 vector arm) over the HNSW index built by ISSUE-023; the uniform candidate filters (confidence floor / not-expired / not-superseded, applied to *both* arms, with the `system_pointer` admission rule); the **clearance + visibility filter that runs before ranking, never after** (realising C1's rule as the retrieval mechanism, plus the optional agent-scope narrowing predicate on the agent path); the weighted ranking formula + procedural boost + top-N trim; injection as a type-tagged Business Context section with Restricted-never-auto-injected; and the answer-mode / Retrieval-Sufficiency `[Building]`-flag **signal emission** (C2 owns the signals; the pill *rendering* is C8). This slice owns the composition of the clearance predicate with pgvector ranking on the hot path (the NFR-PERF.001/003/006 posture).
**Out:** The HNSW index, embed-on-write, and `ef_search` dial that this path *reads* — ISSUE-023 (C2 VEC). The clearance/visibility/Restricted **model + RLS enforcement** whose rule this path realises — ISSUE-019/020 (C1 CLR/RST/RLS). The pill **rendering** and the Inferred-vs-Cited decision that depends on whether the agent used memory — ISSUE-062/053 / chat surface (C8). Layer-3 prompt-injection scoping / volume bounds that *call* retrieval — ISSUE-045 (C4 INJ). The per-agent `memory_scope` definition the agent-scope predicate consumes — ISSUE-063 (C8 SCO). Maturity computation the `[Building]` flag reads — ISSUE-030 (C2 MAT). The memory-navigation surface that renders retrieval output — ISSUE-031.

## 8. Build order within the slice
1. **Entity extraction (FR-2.RET.001):** parse the incoming task to identify its entities, resolving against `DATA-entities` via **FR-2.ENT.005** (the `external_refs`-first, then deterministic name/type match, per OD-033 — this is the "ISSUE-022 resolution helper": ISSUE-022 builds the code that realises FR-2.ENT.005; its interface is that FR, now in the Context manifest) to seed the keyword arm. A not-yet-known entity → no keyword hit, vector arm still runs. → AC-2.RET.001.1.
2. **Dual search (FR-2.RET.002):** keyword arm filtered by entity membership; vector arm embeds the task text and pulls the top-~20 by cosine over the ISSUE-023 HNSW index at the configured `ef_search`; union the two candidate sets before filtering/ranking. → AC-2.RET.002.1.
3. **Candidate filters (FR-2.RET.003):** apply confidence ≥ `CFG-retrieval_confidence_threshold` (0.7), `expires_at` null-or-future, and `superseded_by` null **uniformly to both arms** (OD-035 — else a stale memory re-enters via semantic similarity); admit `system_pointer` (unscored) on its own rule. → AC-2.RET.003.1.
4. **Clearance + visibility filter BEFORE ranking (FR-2.RET.004):** drop candidates failing the requester's visibility **or** entity-type-scoped sensitivity clearance (FR-1.CLR.004/006) *before* any ranking — never rank-then-strip. Human/session path composes with RLS (ISSUE-020); the agent `service_role` path uses harness clearance and additionally accepts the optional **agent-scope predicate** (`memory_scope`, ISSUE-063) that narrows *within* clearance, never widens it (OD-081). This is the AF-067 hot-path composition — the `(select …)` initPlan predicate over the batch. → AC-2.RET.004.1, AC-2.RET.004.2; AC-NFR-PERF.001.1/.001.2.
5. **Rank + trim (FR-2.RET.005):** normalize each raw signal to `[0,1]` **per OD-169** before weighting — **recency** = `0.5 ^ (age_days / CFG-rank_recency_half_life_days[90])` over `created_at`; **confidence** used directly (already 0–1; `system_pointer` unscored → excluded from this term); **entity-match** = Jaccard overlap of the task's resolved entities (step 1) against the candidate's `entity_ids`; **vector-similarity** = cosine mapped via `(cosine+1)/2` — then score by the LIVE compound `ranking_weights` (0.3/0.3/0.2/0.2), apply the ×`CFG-procedural_boost` (1.2) to procedural memories, sort, take the top `CFG-memories_injected_per_task` (7). (Normalization *shapes* fixed by OD-169; the half-life + weights are AF-002-tuned; the scoring SQL body is a Phase-4 build artifact.) → AC-2.RET.005.1; AC-NFR-PERF.006.1.
6. **Inject as Business Context (FR-2.RET.006):** format the selected memories as a type-tagged ([Semantic]/[Episodic]/[Procedural]) Business Context section for Layer-3 (ISSUE-045); enforce Restricted-never-auto-injected (FR-1.RST.003) unconditionally, even for a cleared holder. Retain injected memory IDs for provenance. → AC-2.RET.006.1.
7. **Answer-mode / Sufficiency signals (FR-2.RET.007):** compute Retrieval Sufficiency inline (ADR-002 — a thin threshold over the retrieval signals already produced, not a new engine): the query is **Sufficient** when its touched slots on the primary entity are filled AND retrieval surfaced them above the relevance×confidence bar, else it falls back to pure top-k relevance×confidence — the numeric "thin" bar is `CFG-retrieval_sufficiency_threshold` (**LIVE, 0.6**). Read touched-entity Maturity (ISSUE-030) and emit (a) injected-memory provenance for the Cited pill and (b) the Sufficiency verdict + `[Building]` flag: **low Sufficiency AND primary-entity Maturity < `CFG-cold_start_proactive_threshold` (50%; = ADR-002's `proactive_threshold`) → `[Building]`; low Sufficiency + mature entity → plain Unknown** (ADR-002 §3/§4). Rendering is C8. → AC-2.RET.007.1, AC-2.RET.007.2.
8. **Observability hook:** retrieval candidate counts, ranking inputs/outputs sampled for the AF-002 EVAL, and the answer-mode distribution metric (the thin-coverage signal) to `event_log`; Personal/Restricted candidate access to `access_audit`.
9. **Tests to the ACs** (below).
