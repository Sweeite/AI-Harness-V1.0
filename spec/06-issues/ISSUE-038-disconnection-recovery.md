## 1. Goal (one line)
Build the connector disconnection + recovery lifecycle — detect a dead/revoked connector and classify it system-wide vs individual, surface it (non-dismissible modal to admins, banner to standard users), auto-resume the tasks it paused on reconnect (re-checking authorization before any consequential side effect and persisting the paused set across restarts), escalate to Super Admin if unresolved past the configurable window (clock persisted across restarts), and emit the connector health-panel data + expiry/degraded/unresolved alerts — so a connector failure is always seen, never silently drops a task, and always resolves or escalates.

## 2. Scope — in / out
**In:** The complete `DSC` disconnection/recovery flow (C3 FR-3.DSC.001–006): (a) **detection + classification** — the runtime marks a connector `degraded` on a failed call / dead refresh (from FR-3.TOK.004) / revocation event and classifies the outage as system-wide (app uninstalled, refresh dead) vs individual (one user's grant lapsed), which drives surfacing and reconnect authority; (b) **surfacing behaviour + authority** — a non-dismissible modal (with a one-click re-auth) to Admin/Super-Admin who can reconnect, an informational banner to standard users, with reconnect authority = Admin/Super-Admin for system-wide and the affected user for individual lapses (C3 *defines* the behaviour + authority + non-dismissibility; C7 renders the pixels — see Out); (c) **auto-resume** — on reconnect the tasks paused by the disconnection resume from where they paused with a pause/resume audit trail, the paused-task set is **persisted and recovered across a runtime restart** (never silently abandoned), and a resume whose next step is a consequential external side effect **re-checks authorization (consumes C1 FR-1.RLS.007) before executing** — a revoked authorization halts-and-escalates rather than acting; (d) **escalation** — an unresolved disconnection past `CFG-connector_disconnection_escalation_window` (default 24h) escalates to Super Admin with a loud alert, the escalation clock runs from the detection timestamp and is **persisted so it is honoured across restarts** and fires even if the modal was deferred; (e) **health-panel data emission** — per connector, emit status / last-successful-call / token-expiry countdown / rate-limit headroom for the dashboard to render, never exposing token material, showing missing/stale health data as a warning not a blank; (f) **alerts** — refresh-token expiring within `CFG-token_expiry_alert_days` (default 7) emails the connector owner, degradation raises the modal, unresolved past window escalates, and an alert that cannot be delivered is itself surfaced (not silent).

**Out:** The **token-lifecycle mechanism** (Vault custody, the 3-layer proactive/reactive/dead-token refresh, atomic rotate+persist, `credential_state` transitions to `degraded`/`dead`) → **ISSUE-033** (FR-3.TOK.*). This slice *consumes* FR-3.TOK.004's dead-token → degraded signal and the one-click re-auth it exposes; it does not implement token refresh. The **connector contract + shared runtime + tool registry** (FR-3.CONN/REG) → **ISSUE-032** (this slice runs inside that runtime). The **trigger/liveness path** (watch re-arm, event-gap reconcile) → **ISSUE-037** (FR-3.TRIG.005/006 *enter* this disconnection flow via FR-3.DSC.001, but the re-arm/reconcile logic is ISSUE-037's). The **task_queue status machine, pause/resume state, and the persisted-paused-task + DLQ recovery infra** → **C5 (ISSUE-048 task_queue status machine, ISSUE-052 Inngest execution + DLQ)** — this slice *drives* the pause on disconnect and the resume-with-re-check on reconnect and *records* the audit, but the durable task state + restart-recovery substrate it relies on is C5's. The **RLS `service_role` originating-user binding + the FR-1.RLS.007 mid-task-revocation re-check helper** → **ISSUE-020** (this slice *calls* that helper at the resume side-effect boundary; it does not author the RLS policy). The **actual rendering** of the modal/banner, the connector health panel, and the escalation/expiry alerts → **C7 / ISSUE-075 (alerting + notification centre) + ISSUE-078 (ops dashboards, surface-05/06)** — this slice *emits* the health data + alert triggers + recipients and *defines* the modal's non-dismissibility and authority, but does not build the dashboard surface. The **per-connector revocation signals** (Slack `app_uninstalled`/`tokens_revoked`, Google revocation triggers) → the connector-instance issues **ISSUE-039/040/041**, which feed detection signals into FR-3.DSC.001's classification defined here. OD-010 compensation for partial external-write chains on a paused/resumed task → **C5/C6/C8** carry-in (not this slice).

2. **Config:** register `CFG-connector_disconnection_escalation_window` (24h) and `CFG-token_expiry_alert_days` (7) with the ISSUE-010 config store + their edit-classes.
3. **Detection + classification (FR-3.DSC.001):** on a failed call / FR-3.TOK.004 dead-refresh / revocation event, mark the connector `degraded` and classify system-wide vs individual; persist a durable disconnection-state record with the **detection timestamp** (this timestamp is load-bearing for DSC.004.2). Emit `event_log`.
4. **Surfacing behaviour + authority (FR-3.DSC.002):** decide modal-vs-banner and reconnect authority from the classification (system-wide → Admin/Super-Admin modal; standard user → banner; individual lapse → affected user); enforce non-dismissibility (must resolve or explicitly defer — deferral does not stop the escalation clock in step 6). Emit surfacing/acknowledgement to `event_log`; hand the render contract to C7 (ISSUE-075/078).
5. **Pause on disconnect → auto-resume on reconnect (FR-3.DSC.003):** pause the dependent tasks (drive C5/ISSUE-048's status machine); persist the paused-task set so it survives a runtime restart (AC-3.DSC.003.3); on reconnect resume from the pause point and write pause+resume to `audit` (AC-3.DSC.003.1).
6. **Resume-time authorization re-check (FR-3.DSC.003 AC-.3.2, consumes C1 FR-1.RLS.007):** before a resumed task's next *consequential external side effect*, re-check the originating user is still active and the relied-on authorization is in force via ISSUE-020's helper; a revoked/deactivated authorization **halts-and-escalates** rather than acting.
7. **Escalation (FR-3.DSC.004):** run the escalation timer from the step-3 detection timestamp against `CFG-connector_disconnection_escalation_window`; persist the clock so it is honoured across restarts (AC-3.DSC.004.2) and fires even if the modal was deferred; on elapse raise a loud Super Admin escalation (AC-3.DSC.004.1) — satisfies NFR-OBS.007 (escalate + persist, never silently park).
8. **Health-panel data emission (FR-3.DSC.005):** emit per-connector status / last-successful-call / token-expiry countdown (from `connector_credentials.expires_at`) / rate headroom (from `rate_limit_tracker`, FR-3.RL.001), **never token material**; render missing/stale health data as a warning, not a blank. (C7/ISSUE-078 renders.)
9. **Alerts (FR-3.DSC.006):** token expiring within `CFG-token_expiry_alert_days` → email the connector owner; degradation → raise the modal (step 4); unresolved past window → the step-7 escalation; an alert that cannot be delivered is itself surfaced (not silent). Emit to `event_log`; hand delivery to C7/ISSUE-075.
10. **Observability hooks:** disconnection detected/classified, surfaced/acknowledged, pause/resume (audit), resume-time revocation halt (audit + escalate), escalation raised, alert sent / alert-delivery-failed — all to `event_log`/`audit`.
11. **Tests to the AC** (see Verification).

**Integration note (bundled FRs):** DSC.001–006 are one slice because they are one lifecycle with a single #3-invariant running through it — *a connector failure is always seen and never silently drops a task*. Detection (001) is meaningless unless it surfaces to someone who can act (002); surfacing is incomplete unless the work it stalled resumes or escalates (003/004); resume is unsafe unless it re-checks authorization at the side-effect boundary (003.2, the ADR-006 containment point) and durable unless the paused set + escalation clock survive a restart (003.3 / 004.2, the NFR-OBS.007 keystone); and the whole loop is invisible unless the health panel (005) and alerts (006) expose it. Splitting these would leave a window where a connector is down but no one is prompted, or a paused task is quietly abandoned on a restart, or a resumed task acts on a revoked grant — each a silent #3 failure. The two persistence requirements (paused-set, escalation-clock) are the load-bearing seam to C5's durable substrate: this slice *owns the policy* (what pauses, when it escalates, who is authorized) but *relies on* ISSUE-048/052 for the durable state that makes "never forgotten across a restart" true.
