# ISSUE-031 — Memory navigation surface

> **Self-sufficiency contract (read this first).** This issue is a *complete, precise build
> order that points into the repo by ID*. It does **not** restate `AC-*` text — that lives in the
> FR and is read there (copying it would create a second source of truth that rots = Rule-0
> violation). A builder with **zero conversation history** must be able to open the files named in
> the Context manifest and build this slice to its Definition of done **without guessing**.

## 1. Goal (one line)
Build **`UI-MEMORY-NAV`** (surface-11) — the human read/browse window into the business brain: the Entity Browser, Entity Detail, Memory Detail, and Memory Search, all clearance-scoped at the row, reusing the retrieval pipeline (ISSUE-025) as browse and rendering the entity/memory/maturity model and the maintenance-lifecycle signals (ISSUE-027) — the leaf that makes stored knowledge *visible* without ever writing it directly.

## 2. Scope — in / out
**In:** The **surface-11** UI slice only — a browser + detail-drawer console reached from the main nav, comprising the four sections defined in `surface-11-memory-nav.md`: **(A) Entity Browser** (clearance-scoped entity grid, type filter over `CFG-entity_types`, per-entity Maturity % + `[Building]` marker, Internal-Org badge, possible-duplicate flag), **(B) Entity Detail** (one entity's memories grouped by type, filled/empty knowledge slots, `external_refs` pointers, related entities), **(C) Memory Detail** (one memory row in full — content/type, provenance, confidence + lifecycle state, visibility × sensitivity, drillable supersede/summary chain, expiry), and **(D) Memory Search** (the dual keyword + vector search exposed for a human, clearance-before-ranking). This slice **owns**: all four sections' data bindings, the five render states each (Loading / Empty / Error / Partial / Offline-stale) with their #1/#2/#3 honesty rules, the read-only reflection of the four CFG values, the `access_audit` write on Confidential/Restricted/Internal-Org view, and the wiring of every *mutation control* (correct/re-tag, merge, verify/flag, reveal-Restricted, initiate-erasure) to its owning path — the browser issues **no direct `UPDATE`** (ADR-004). Entry is **any authenticated user**; the row-level clearance/visibility/Restricted RLS is the gate (OD-145 — **no browse PERM node is minted**). The two always-loud notification banners (`AC-7.ALR.008.2`, `AC-7.ALR.009.1`) are pinned. Mobile: single-column collapse, mutations best-effort/discouraged (surface-12 owns detailed mobile).
**Out:** The retrieval/dual-search/clearance-before-ranking **engine** the search bar and browser filter *call into* — ISSUE-025 (C2 RET). The maintenance-lifecycle **jobs** (confidence decay, supersede, summarise, erosion, expiry) whose *outputs* the browser renders — ISSUE-027 (C2 MNT lifecycle). The **sole-writer** validate-and-commit path a correction/re-tag routes to — ISSUE-024 (C2 WRT). The **entity-merge / disambiguation** writer-side operation a duplicate flag routes to — ISSUE-022 (C2 ENT) / OD-033. The **compliance-erasure transitive cascade + verify-before-done** the "initiate erasure" control starts — ISSUE-029 (C2 MNT.017 memory-side) → ISSUE-082 (C10 individual right-to-erasure). The **conflict / consolidation review decision** a flagged item links out to — ISSUE-028 / surface-03. The **entity-type + expected-slot config editors** (values reflected read-only here) — surface-01 / ISSUE-086 (OD-147). The **answer-mode pill** rendering — C4/C8 (chat/activity surfaces, not here). The **RLS clearance predicate** itself — ISSUE-009/020 (C1 RLS).

## 8. Build order within the slice
1. **Read data-access layer (clearance-scoped, RLS-backstopped):** wire entity + memory reads through the C1 row-level clearance/visibility/Restricted RLS (FR-1.RLS.003) on the **human path** (no `service_role`, ADR-006) — the clearance predicate must compose with the entity filter within budget (AF-067). Confirm `access_audit` fires on any Confidential/Restricted/Internal-Org read and that its write cannot fail silently. **No `client_slug`.**
2. **Section A — Entity Browser (landing):** the clearance-scoped entity grid off DATA-entities; type filter reflecting CFG-entity_types; per-entity Maturity % (`entities.maturity`) with CFG-cold_start_full_threshold context; `[Building]` marker (CFG-retrieval_sufficiency_threshold); Internal-Org badge; duplicate-cluster flag (FR-2.MNT.010, the AF-082 backstop). Wire all five states — never render an empty grid as a false-empty brain; distinguish cold-start vs zero-cleared-scope on Empty; mark stale Maturity "as-of HH:MM", never a false 100%. → AC-2.ENT.001.1/002.1/002.2/003.1/003.2/005.1/005.2, AC-2.MAT.002.1/003.1.
3. **Section B — Entity Detail drawer:** header from DATA-entities (`external_refs`); memories grouped by type (DATA-memories where entity ∈ `entity_ids`), clearance-scoped **before display**; filled/empty knowledge slots from the expected-slots store (CFG-expected_slots); related entities. Distinguish "no memories you're cleared to see" from "this entity has no memories"; a missing slot readout never renders as 100% Maturity. → AC-2.MEM.001.1, AC-2.ENT.001.1/004.1, AC-2.MAT.001.1.
4. **Section C — Memory Detail:** the full row (content/type, provenance source/source_ref, confidence + lifecycle state with contradiction flagged not hidden, visibility × sensitivity, expiry) + the **drillable supersede/summary chain** (`superseded_by`, both directions) — a chain read failure shows "history couldn't load", never an empty chain implying no prior versions; provenance never silently reads "verified" when unconfirmed. → AC-2.MEM.002.2, AC-2.TAG.001.1/002.1/002.2/003.1, AC-2.MNT.001.1/001.2/004.1/006.1/007.1.
5. **Section D — Memory Search:** the dual keyword + vector search (FR-2.RET.002) reusing ISSUE-025's engine, with the **identical clearance filter running before ranking** (FR-2.RET.004) — out-of-clearance excluded first, never ranked-then-hidden; if one arm fails, show the returned arm + an explicit "other results unavailable", never a half-set as complete. → AC-2.RET.002.1/004.1/006.1/007.1/007.2.
6. **Mutation-control wiring (read-first — the browser issues NO direct `UPDATE`, ADR-004):** correct/re-tag → sole-writer validate-and-commit API (ISSUE-024, `PERM-memory.write`); merge/disambiguate → entity-merge path (ISSUE-022 / OD-033, `PERM-memory.write`); verify/flag → logged confidence-signal feedback (FR-2.MNT.016, cleared viewer); reveal-Restricted → explicit **audited** access (matching grant, FR-1.RST.001, `access_audit`); initiate-erasure → the audited compliance-erasure workflow (ISSUE-029/082, `PERM-memory.delete` + erasure gate); conflicted/consolidation item → surface-03 link-out. All mutation nodes default-deny; mutations disabled offline. → AC-2.MNT.010.1/016.1/017.1, AC-2.RET.006.1.
7. **Chrome + a11y + mobile:** pin the two always-loud banners (AC-7.ALR.008.2 / AC-7.ALR.009.1); apply the NFR-A11Y.001 baseline (keyboard nav, contrast, semantic markup, labelled controls, no colour-only status); single-column mobile collapse with mutations best-effort/discouraged (surface-12). → AC-NFR-A11Y.001.1/001.2.
