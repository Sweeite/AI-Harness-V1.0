# ISSUE-024 — Memory write / sole-writer path

> **Self-sufficiency contract (read this first).** This issue is a *complete, precise build
> order that points into the repo by ID*. It does **not** restate `AC-*` text — that lives in the
> FR and is read there (copying it would create a second source of truth that rots = Rule-0
> violation). A builder with **zero conversation history** must be able to open the files named in
> the Context manifest and build this slice to its Definition of done **without guessing**.

## 1. Goal (one line)
Build the single, sole-writer memory write path — the Memory Agent's `service_role` writer that runs one Sonnet call unlocked, contradiction-checks against the similar set, then commits each write under a per-entity advisory lock with watermark re-validation, idempotency, CAS-supersede, mid-task-revocation halt, and embedding-failure halt — so every memory row is created safely, exactly once, and never on a stale or unauthorized snapshot.

## 2. Scope — in / out
**In:** The complete `WRT` write flow (C2 FR-2.WRT.001–007): the sole-writer invariant (no other agent/pipeline touches `memories`/`entities` directly); the pre-write contradiction check that classifies no/soft/hard conflict and writes-then-CAS-supersedes on soft, quarantines on hard; the single Sonnet memory-writer call that emits one-or-more typed, entity-linked memories with confidence/sensitivity/expiry; the golden-rule `system_pointer` path (pointer + enrichment, never a copied source record); source-typed confidence assignment; the ADR-004 optimistic **validate-and-commit** (LLM unlocked → short txn under sorted per-entity advisory locks → watermark re-check → idempotency-keyed insert + CAS-supersede), including the mid-task-revocation re-check at the commit boundary (realizing C1 FR-1.RLS.007); and the embedding-failure halt that routes the source event to a retry queue rather than storing a null/invalid vector. This slice makes the `memories`/`entities` tables *writable* through exactly one governed code path.

**Out:** Filter 1 (relevance / the Haiku selective-writing gate, FR-2.ING.001), Filter 2 (sensitivity), the ingestion queue and the three pipelines (FR-2.ING.*) → **ISSUE-026** (the write path *is invoked by* ingestion but does not define the ingestion filters/queue; ingestion routes *through* this writer per FR-2.ING.010). Embeddings, the HNSW index, and the re-embedding migration (FR-2.VEC.*) → **ISSUE-023** (this slice *calls* the embedder and enforces FR-2.WRT.007's halt-on-failure, but does not build the vector index or migration). Retrieval/ranking (FR-2.RET.*) → **ISSUE-025**. The **hard-conflict review surface** (`memory_conflicts` quarantine UI on surface-03) + Personal-tier consolidation approval → **ISSUE-028** (this slice *inserts* the quarantine row and sets its escalation clock; ISSUE-028 renders/actions it). The daily supersede + weekly merge maintenance backstops, decay, and transitive erasure (FR-2.MNT.*) → **ISSUE-027/028/029**. The memory/entity model + schema tables + sensitivity/visibility tagging semantics (FR-2.MEM/ENT/TAG) → **ISSUE-022** (this slice consumes those columns and the entity-resolution helper; it does not define them). The RLS enforcement + `service_role` originating-user binding + mid-task-revocation *mechanism* (FR-1.RLS.004/007) → **ISSUE-020** (this slice *consumes* that mechanism at the commit boundary; it does not author the RLS policies or the revocation re-check helper).

## 8. Build order within the slice
1. **Migration prerequisites (verify, don't re-create):** confirm ISSUE-022's `memories`/`entities` migration landed with the `unique(idempotency_key)` constraint, the `content_hash`/`idempotency_key`/`updated_at` columns, the `(entity_ids, updated_at)` watermark index, and the `check(cardinality(entity_ids) >= 1)` / `check(source='system_pointer' or confidence is not null)` constraints (schema §3). Confirm `memory_conflicts` exists. If any is absent, it is an ISSUE-022 gap — flag, do not silently add it here.
2. **Config:** register `CFG-rate_limit_memory_writes_per_minute` (30, LIVE) and `CFG-memory_write_serialization` (`per_entity`) with their edit-classes on the ISSUE-010 config store; enforce the writer rate cap (never unlimited, AC-NFR-COST.008.2).
3. **Sole-writer boundary (FR-2.WRT.001):** implement the single `service_role` writer entry point; ensure no other code path inserts/updates `memories`/`entities` (a negative invariant — enforce by making the writer the only holder of write privilege / the only module importing the write API). A human `PERM-memory.write` action invokes this same path with source `human_verified`.
4. **Contradiction check (FR-2.WRT.002):** pull the 3–5 most similar existing memories (uses ISSUE-023's vector arm read); classify no/soft/hard. This runs **unlocked** (part of the slow path).
5. **Sonnet writer (FR-2.WRT.003) + golden rule (FR-2.WRT.004) + confidence (FR-2.WRT.005):** exactly one Sonnet call drafts one-or-more typed, entity-linked memories (resolving entities via ISSUE-022's resolver); system-of-record data becomes a `system_pointer` (`source_ref` + enrichment, no copy — AC-NFR-CMP.002.1); assign source-typed confidence. Cost shape = ≤1 Sonnet wrapped in ≤3 Haiku (AC-NFR-COST.008.1).
6. **Embedding-or-halt (FR-2.WRT.007):** embed the drafted memory via ISSUE-023's embedder; on failure/timeout/degenerate vector, **halt the commit** and enqueue the source event to the write-failure/retry queue with an alert — never proceed to step 7 with a null/invalid embedding.
7. **Validate-and-commit (FR-2.WRT.006, ADR-004 §3):** open the short txn; acquire sorted per-entity advisory locks (`pg_advisory_xact_lock(hashtext(eid))` over sorted `entity_ids`); re-read the per-entity watermark; if unchanged commit as-is, else re-run only the **cheap DB** contradiction check (no second Sonnet call) and commit/re-target/bounce. Insert idempotency-keyed (`ON CONFLICT DO NOTHING`); CAS-supersede on soft conflict (`WHERE superseded_by IS NULL`); on hard conflict insert the `memory_conflicts` quarantine row (not the live set) and set its escalation clock.
8. **Mid-task-revocation halt (FR-2.WRT.006 AC-.6.3, consumes C1 FR-1.RLS.007):** at the commit txn boundary, re-check that the originating user is still active and the relied-on clearance/Restricted grant is still in force (via ISSUE-020's helper); if revoked/deactivated mid-write, halt + quarantine the pending memory and write an `audit` row — a benign session-expiry alone does not halt (C0 FR-0.SESS.006).
9. **Observability hooks:** emit soft-supersede / hard-conflict-flag / overdue-escalation `event_log`; embedding-failure `event_log` + alert; mid-task-revocation halt → `audit` + quarantine; agent-path Personal/Restricted access at write → `access_audit` (FR-1.AUD.001, AF-081 coverage).
10. **Tests to the AC** (see Verification).

**Integration note (bundled FRs):** WRT.001–007 are one slice because they are one code path — the sole-writer boundary (001) makes the contradiction check (002), the Sonnet writer (003/004/005), the embedding gate (007), and the concurrency-safe commit (006) trustworthy *only if nothing bypasses them*. The ADR-004 shape is load-bearing across the bundle: the LLM work in steps 4–6 runs **unlocked**, and step 7's short locked txn is the single point where correctness (watermark), idempotency, supersession, and mid-task authorization are all re-validated together. Splitting these would create a window where one guarantee holds without the others. The cheap on-race re-check is intentionally lexical/DB-level (no second Sonnet call); a *semantically*-contradicting racing write is caught not here but by the daily supersede backstop (FR-2.MNT.006, ISSUE-027) within ≤1 day — bounded and surfaced, not silent.

## 9. Verification (how DoD is proven)
- **Unit / integration** (per `spec/05-non-functional/test-strategy.md`): sole-writer invariant — no non-writer path mutates `memories` (AC-2.WRT.001.1); soft-conflict CAS-supersede leaves the chain intact, hard-conflict quarantines + escalates (AC-2.WRT.002.1/.2/.3); one Sonnet call emits multiple typed entity-linked memories (AC-2.WRT.003.1); system-of-record data stored as a pointer with no copy (AC-2.WRT.004.1, AC-NFR-CMP.002.1); source-typed confidence bands (AC-2.WRT.005.1); embedding-failure halt + retry-enqueue + alert, degenerate-vector rejection (AC-2.WRT.007.1/.2); write-path cost shape ≤1 Sonnet + ≤3 Haiku and the 30/min Sonnet cap (AC-NFR-COST.008.1/.2).
- **Concurrency / load** (the AF gates): same-entity concurrent writes → at most one wins per watermark re-check, no duplicate / lost-supersede (AC-2.WRT.006.1); disjoint-entity writes never block (AC-2.WRT.006.2); mid-write deactivation/clearance-revoke halts + quarantines at the commit txn, benign expiry does not (AC-2.WRT.006.3). **AF-061** (window closed, no livelock) and **AF-062** (locks don't bottleneck / deadlock-free under fan-out) must be GREEN, and **AF-063** confirmed (or the advisory-lock-alone fallback applied), before the AC-2.WRT.006.* criteria move to `Verified`.
- **AC → Verified path:** each listed AC-* moves to `Verified` when its test layer passes on CI atop the ISSUE-022 model + ISSUE-020 RLS/service_role baseline; the slice is `done` only when every AC in §4 is `Verified` and AF-061/062/063 are resolved (GREEN or fallback applied).
