## Gap summary (the review checklist)

1. **Settings model is undefined.** ~85-90% of config keys have no stated editor-role,
   UI surface, or live/reload behaviour. This is the single biggest missing artifact.
2. **No config↔UI mapping.** `/tune` (L3904), self-improvement panel, and the config
   file never reference each other. Which keys are `/tune`-able? Where does each live?
3. **Surface visibility mostly inferred.** Only the RBAC matrix (L502-616) states roles
   per dashboard; individual panels within dashboards have no explicit role gating.
4. **Real-time vs polled defined for only 8 surfaces** (L3068-3179). Every modal, banner,
   and the chat interface is unspecified.
5. **No validation rules** for any config value (min/max, allowed enums beyond a couple).
6. **Coverage % undefined** — drives cold_start.* thresholds and the [Building] pill but
   is never defined as a metric.
7. **No empty/error/loading state defined per panel** — the doc gives generic patterns
   (L1242-1333) but doesn't map them to specific surfaces.
